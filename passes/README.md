# passes

Per-pass work logs for the Adams Cosby CRM build. Each pass is one focused improvement sprint.

## Naming Convention

```
pass-{letter}-{kebab-description}.md
```

Examples:
- `pass-a-list-screen-layout.md`
- `pass-g-kpi-filters-back-button.md`

## 5-Section Self-Critique Format

Every pass log ends with a mandatory self-critique block:

```markdown
## Self-Critique

### 1. What worked
[What went right in this pass]

### 2. What didn't work
[Failures, dead ends, or approaches abandoned]

### 3. Risk flags
[Anything that could break in a future pass or in production]

### 4. Open items
[Decisions deferred or work left incomplete]

### 5. Next pass recommendation
[Suggested focus and approach for the next pass]
```
